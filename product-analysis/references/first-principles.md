# First Principles: 3-Step Product Deconstruction

## The Method

Three steps. One output: the Core Anchor — a single sentence that captures the product's essential truth.

### Step 1: Core Need

What fundamental human desire does this product address?

**Rules:**
- State it in terms a person from any century would understand
- If your answer contains the product's name or any technology, go deeper
- Test: "What would satisfy this need if technology didn't exist?"

**Bad**: "People need a photo-sharing app" → describes a solution
**Good**: "People need to feel connected to others when apart" → describes the desire

### Step 2: Ideal Form

Given the core need + today's constraints (technology, economics, cognition, regulation), what is the most natural form?

Don't list features. Describe:
- What the user **feels** when the product is working perfectly
- What **friction disappears**
- What the product **refuses to do** (negative space)

The ideal form is not utopian — it respects real constraints. But it doesn't carry historical baggage.

### Step 3: The Gap → Core Anchor

Compare ideal form to current reality. Find the single most important tension.

State it as one of:
- **"X 的本质矛盾是 ___"**
- **"X is ___ pretending to be ___"**
- **"X chose ___ at the cost of ___"**

This is the Core Anchor. It drives the entire report. If you can't state it in one crisp sentence, your analysis isn't deep enough — go back to Step 1.

## Common Traps

1. **Mistaking features for needs** — "Users want dark mode" is not a need. Eye comfort in low light is.
2. **Ignoring dead constraints** — Many products carry decisions from constraints that no longer exist. Identify these.
3. **Over-indexing on current users** — They self-selected for the current product. Consider non-users and churned users.
4. **Being too abstract** — If your Core Anchor could apply to any product in the category, it's not specific enough.
